Variables:
- A variable is a name that stores a value
- Python uses dynamic typing
  - The type is determined at runtime
- You do not need to declare a type in advance

- Then assigns the result to the left-hand side

Meaning of the dot (.) in Python

Dot with modules:

  • Using . after a module means:
    • Go inside the module
    • Access any options (functions, variables) available in that module

  • math. work with what is inside the math module

Dot with objects (variables):

  • Using . after an object means:
    • Look at what options the class of that object provides

Important idea:

  • Every object in Python comes from a class
  • The dot shows what that class allows you to work with

Summary:

  • module . option options inside a module
  • object . method methods defined by the objects class

Looping over data structures

  • Looping over data structures (like lists, dictionaries, etc.) will be covered later

When to use for vs while

  • for is used when:

    • We want to write a loop over a set
    • Or when we know how many times we want to repeat a certain task

  • while is used when:

    • We do not know exactly how many times the work will be repeated
    • But we do know the condition under which it should end
while condition:
    statement
In[82]:
a = 5
while a > 5:
    print("Hi")

never do this:
a = 5
while a > 4:
    print("Hi")

Why should you never do this?

  • The condition a > 4 is always True
  • The value of a never changes
  • So the loop never ends

What would happen if this code ran:

  • "Hi" would be printed forever
  • This creates an infinite loop

Important rule for while loops:

  • Something inside the loop must change the condition
  • Otherwise, the loop will never stop

What does this code do?

  • "Hi" is printed in each iteration
  • The value of a is decreased by 1 every time the loop runs
  • The if statement checks whether a has become negative

What is break?

  • break is a control statement used inside loops
  • When Python encounters break, it immediately exits the loop
  • Execution continues with the code after the loop

Why is break needed here?

  • The loop condition (True) never becomes False on its own
  • The stopping condition is instead defined inside the loop
  • When a < 0 becomes True, break stops the loop

Key idea

  • break allows us to terminate a loop based on a condition
  • This is useful when the stopping logic cannot be written directly in the while condition

Important note about break

  • break can appear anywhere inside a loop
  • As soon as Python encounters break, it immediately exits the loop
  • No further statements inside the loop are executed

break in a for loop

  • break can also be used inside a for loop
  • As soon as break is encountered, the loop stops immediately
  • The loop does not continue to the next iteration

12. Dictionary

Why lists are not good for tagged data

  • Lists are good for ordered data
  • But lists are not suitable for tagged or labeled data

Example of tagged data:

  • city id, name, population, rainfall, average age

Problem with lists:

  • Values are stored only by position
  • It is hard to remember what each position means
  • Data is not easily trackable or readable

What is a dictionary?

  • A dictionary stores data as keyvalue pairs
  • Each value is accessed using a key, not an index
  • Dictionaries are designed for labeled (tagged) data
  • Dictionary keys must be unique
    • A key cannot be repeated
  • Keys are used to identify values

Key types:

  • Keys should be of type:
    • str
    • int

Important notes:

  • If a key is repeated, the last value overwrites the previous one
  • Keys are used to access values, not positions

13. Set

What is a set?

  • A set is a collection of unique elements
  • Sets are unordered
  • Duplicate values are automatically removed

Important properties:

  • No repeated elements
  • Order does not matter
  • Elements are accessed by membership, not by index

Why use a set?

  • When you only care about whether something exists
  • When you want to remove duplicates
  • When order is not important

Set as a mathematical concept

  • A set in Python is very similar to a mathematical set
  • You can perform common set operations on sets, such as:
    • union ()
    • intersection ()
    • difference ()

Important idea:

  • Sets are designed for set logic, not indexing or ordering

14. Tuple

What is a tuple?

  • A tuple is an ordered collection of elements
  • Tuples are immutable
    • meaning their content cannot be changed after creation

Important properties:

  • Order matters
  • Elements are accessed using indexing
  • Values cannot be modified once the tuple is created

Why use a tuple?

  • When you want to store a group of values that should not change
  • When data represents a fixed structure
  • Tuples are often used for:
    • coordinates
    • settings
    • return values from functions

15. Functions

def func_name(optional_arguments):
    statement
    ...
    return optional_value

What is a function?

  • A function is a reusable block of code that performs a specific task
  • Instead of writing the same logic again and again, we write it once and call it when needed

1) Function definition (def)

  • def is the keyword used to define a function
  • func_name is the name of the function (you choose it)
  • optional_arguments are inputs to the function (also called parameters)
  • After the line ends with :, the function body must be indented

2) Arguments (inputs)

  • Arguments are the values the function receives to work with
  • A function can have:
    • no arguments
    • one argument
    • multiple arguments

Example:

  • In power2(a), the function takes one input called a

3) The function body

  • The indented lines are the steps the function performs
  • The code inside the function runs only when the function is called

4) return (output)

  • return sends a value back to the place where the function was called
  • When Python reaches return, the function stops immediately
  • If a function has no return, it returns None by default
